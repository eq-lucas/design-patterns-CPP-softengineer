# Anotações de Estudo e Aprofundamento

Esta pasta contém materiais complementares de estudo desenvolvidos durante a elaboração deste projeto. O objetivo é aprofundar o entendimento teórico dos Padrões de Projeto para além das implementaçã, codigos auxiliares e comentarios, para compreensão

## Exemplo de conteúdo:

* **`strategy.CPP`**: Implementação didática do padrão Strategy em **C++**.
    * **Objetivo**: Compreender como o padrão funciona em baixo nível, lidando manualmente com alocação de memória, ponteiros, herança e tabelas virtuais (`vtable`).
    * **Diferencial**: O código contém comentários extensivos explicando conceitos de Polimorfismo a questao do virtual, e ponteiros, apontarem para variaveis ( classes que nao sao do mesmo tipo declarado do ponteiro)...
## Metodologia e Ferramentas

Os códigos e explicações contidos nesta pasta foram gerados e refinados de forma colaborativa:
* **Base Teórica**: [Refactoring Guru](https://refactoring.guru/pt-br/design-patterns).
* **Refinamento e Explicação**: Uso da LLM **Google Gemini** para traduzir conceitos complexos de C++ Moderno para C++ Clássico e gerar comentários didáticos linha a linha, além de ajudar no discernimento de design patterns, ficando mais facil a busca pelos ensinamentos.
